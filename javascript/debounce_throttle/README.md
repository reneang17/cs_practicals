# Debounce & Throttle Exercise

## Objective
Implement custom `debounce()` and `throttle()` utility functions from scratch. These are staple performance-optimization functions in Javascript, frequently asked about in interviews to test your knowledge of **Closures**, `setTimeout`, and `this` context binding.

- **Debounce**: Ensures a function is not called again until a certain amount of time has passed without it being called. (Useful for typing search bars).
- **Throttle**: Ensures a function is only called at most once in a specified time period. (Useful for scroll or resize events).

## Instructions
1. Open `index.js` and implement `debounce` and `throttle` based on the provided skeleton.
2. Run `node demo.js` to see a simulated timeline of your functions executing. The demo will tell you if the timing is correct.
3. If you get stuck, reference `solution.js` for an example implementation.
